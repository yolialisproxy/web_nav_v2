# Acceptance Criteria Specification (Acceptance_Criteria.md)
Version: 1.0
Project: WebNav V2

---

## 🔹 定义「完成」
任何宣称「完成」的构建，必须 100% 通过下面所有检查项。
没有例外，没有「差不多」，没有「以后再修」。

---

## 🟢 强制通过项 (Blocker)
### 1. 部署兼容性
✅ 在 GitHub Pages 子路径下零修改正常运行
✅ 所有资源没有 404 错误
✅ 刷新页面不会白屏
✅ 前进后退按钮正常工作

### 2. 性能指标
✅ 首次加载完成时间 < 1.2秒
✅ 分类切换响应时间 < 50ms
✅ 搜索输入响应时间 < 20ms
✅ 侧边栏展开/收缩无卡顿
✅ 滚动帧率稳定 60fps

### 3. 无错误
✅ 浏览器控制台 0 错误
✅ 浏览器控制台 0 警告
✅ 无网络失败请求
✅ 无废弃 API 调用

### 4. 主题系统
✅ 深色/浅色/系统主题正常切换
✅ 刷新页面后主题正确保留
✅ 所有界面元素在两种主题下都清晰可见
✅ 主题切换时无闪屏

---

## 🟡 功能验收项
### 5. 导航系统
✅ 侧边栏正常展开/收缩
✅ 分类点击后正确加载对应网站列表
✅ 同一时间只有一个分类被激活
✅ 滚动时侧边栏保持固定

### 6. 搜索功能
✅ 输入任意字符实时返回结果
✅ 支持标题、描述、标签搜索
✅ 上下键可以导航结果
✅ Enter 键直接打开网站
✅ ESC 键关闭搜索面板

### 7. 交互行为
✅ 所有链接在新标签页打开
✅ 鼠标悬停卡片有正确的视觉反馈
✅ 点击外部区域正确关闭所有打开的面板
✅ 没有意外的页面跳转

---

## 🔴 失败条件
出现以下任意一种情况，视为交付失败：
❌ 需要 npm install / 任何构建步骤才能运行
❌ 引入了任何外部 JS 框架或库
❌ 使用了绝对路径 `/`
❌ 页面加载时有可见的闪屏
❌ 控制台出现任何红色错误
❌ 90% 以下的分类可以正常打开
❌ 搜索需要点击按钮才能执行
