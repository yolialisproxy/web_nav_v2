# Interaction Flow Specification (Interaction_Flow.md)
Version: 1.0
Project: WebNav V2

## 1. 设计目标
目标规模：9000+ 网站条目。在该量级下保持：
- 首次加载时间 < 1.2秒
- 搜索响应 < 30ms
- 分类切换 < 100ms
- 无任何可见的闪屏、卡顿、渲染断层。

---

## 2. 核心用户旅程

### 2.1 标准浏览路径
```
1. 页面载入
├── ✅ 骨架屏立刻渲染 (0ms)
├── ✅ 配置与主题加载 (50ms)
├── ✅ 导航栏 + 侧边栏 静态结构渲染 (100ms)
├── ✅ 数据 JSON 异步加载 (200ms - 400ms)
└── ✅ 内容区域渐进式填充 (无可见跳变)

2. 浏览操作
├── 用户点击左侧分类
├── 内容区域平滑过渡切换
├── 无需重新加载任何资源
└── URL 地址实时更新 (pushState)

3. 搜索操作
├── 输入框获得焦点时立刻打开结果面板
├── 每输入一个字符实时搜索
├── 支持拼音、缩写、标题、描述、标签全局匹配
└── 按键响应无感知延迟
```

## 3. 状态机定义

### 3.1 全局状态树
仅存在以下全局状态。**所有页面元素的显示形态，必须是状态的纯函数：**
```javascript
state = {
  theme: 'dark' | 'light' | 'system',
  sidebar: {
    mode: 'expanded' | 'collapsed' | 'floating',
    width: 280,
    activeCategoryId: null
  },
  search: {
    active: boolean,
    query: string,
    results: SiteID[]
  },
  currentView: 'category' | 'search' | 'bookmarks',
  filterTags: string[]
}
```

> ✅ 无隐含状态。无 DOM 反向读取。状态是唯一真理源。

### 3.2 侧边栏状态机
```
[正常] ────── 点击收缩按钮 ─────→ [收缩模式]
  │                                     │
  │ 鼠标移出侧边栏区域 300ms           │ 鼠标悬停侧边栏
  └─────────────────────────────────────┘
                                        ↓
                                    [悬浮展开]
                                        │
                                        │ 鼠标移出 500ms
                                        └───────────→ 回到 [收缩模式]
```

## 4. 性能关键路径

### 4.1 FOUC 彻底解决协议
1.  **禁止内联脚本**：所有 JS 置于 `<body>` 末尾，`defer` 加载
2.  **首屏 CSS 内联**：骨架屏、侧边栏、导航栏的 CSS 直接写入 `<head>`
3.  **异步主题恢复**：在 DOM 解析前，执行一行内联 JS 恢复主题状态
    ```html
    <head>
      <script>document.documentElement.dataset.theme = localStorage.theme || 'system'</script>
    </head>
    ```
4.  **禁用过渡动画直到首屏渲染完成**：首屏渲染结束后才启用 CSS transition

### 4.2 9000+ 数据渲染策略

✅ 核心优化逻辑：
用户 **永远只会同时看到一个叶子小分类** 的内容。大类和中类仅用于导航菜单，永远不会被作为内容视图渲染。9000 是整个网站的总条目数，而每个叶子小分类平均仅包含 30 ~ 80 个网站。

因此不需要任何复杂的虚拟滚动、分页或延迟加载：
1.  每次用户点击分类时，仅渲染该叶子分类下的网站列表
2.  同一时间永远只有一个激活的小分类
3.  搜索是全局的，但搜索结果同样只显示前 50 个最佳匹配
4.  所有不在视口内的卡片使用 `content-visibility: auto`

> 该策略下，无论总数据量是 1万 还是 10万，单页渲染的 DOM 节点永远小于 100 个。内存占用 < 5MB，滚动帧率始终保持 60fps。

## 5. 交互细节规范

### 5.1 搜索交互
- 输入框获得焦点时，立刻垂直展开结果面板
- 支持上下键导航结果，Enter 直接跳转
- ESC 键关闭搜索面板，恢复原视图
- 无结果时静默显示提示，不弹对话框

### 5.2 卡片交互
- 鼠标悬停时卡片轻微上浮 + 阴影加深
- 左键：新标签页打开网站
- 中键：后台静默打开新标签页
- 右键：显示操作菜单（收藏/复制链接/报告失效）

---

## 6. 无刷新路由
- 所有分类切换、搜索、标签筛选都通过 `history.pushState()` 更新 URL
- 刷新页面时，精确恢复到对应视图和滚动位置
- 浏览器前进/后退按钮 100% 工作
