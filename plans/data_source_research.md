# 欠容填充 - 数据源调研报告

## 候选数据源

1. **Awesome Lists GitHub** - 全领域 (质量: 高)
  链接: https://github.com/sindresorhus/awesome
2. **Public APIs GitHub** - API服务 (质量: 高)
  链接: https://github.com/public-apis/public-apis
3. **AlternativeTo** - 软件替代 (质量: 中)
  链接: https://alternativeto.net
4. **Product Hunt** - 新产品 (质量: 高)
  链接: https://www.producthunt.com

## 填充策略

### 可合并的欠容类（建议合并以减少分类数）
- 总计 60 个只有1个站点的分类可考虑合并
- 合并后可将欠容类从132个减少到约90个

### 需独立填充的欠容类
- 每个需补充 14 个新站点
- 总计约需 788 个新站点

## 执行建议
1. 先合并可合并的分类（立即执行，无需新数据）
2. 批量导入 Awesome Lists 和 Public APIs（高质数据源）
3. 使用 AlternativeTo 补充软件替代类站点
4. 使用 Product Hunt 补充新产品类站点
